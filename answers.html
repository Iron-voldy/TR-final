The Software Development Lifecycle (SDLC) is a systematic process used to design, develop, and maintain software applications efficiently. This structured approach ensures quality, reduces costs, and manages project timelines effectively.

Initially, the process begins with Requirements Gathering, where stakeholders define functional and non-functional specifications. During this phase, project managers conduct meetings with clients to understand their needs and document detailed requirements.

Next, the Testing phase ensures software quality through various testing methods including unit testing, integration testing, and user acceptance testing. Quality assurance teams identify and report bugs for resolution.

After successful testing, the Deployment phase involves installing the software in the production environment and making it available to end users. This includes server configuration and data migration.

Finally, the Maintenance phase provides ongoing support, bug fixes, and feature enhancements. This cyclical phase continues throughout the software's lifecycle, often leading back to requirements gathering for new features or versions.

Solar panel installation is a comprehensive process that transforms buildings into renewable energy generators. This systematic approach ensures safe, efficient, and code-compliant installations that maximize energy production.

Finally, System Commissioning includes testing electrical connections, configuring monitoring systems, and obtaining final inspections. This completion phase ensures the solar installation operates safely and efficiently, providing clean energy for years to come.

Question 3 â€“ Process Description: Cybersecurity Incident Response (25 marks)

Sample High-Scoring Answer:

The Cybersecurity Incident Response Process is a critical framework designed to minimize damage and restore normal operations when security breaches occur. This systematic approach ensures rapid containment while preserving evidence for investigation.

Initially, Incident Detection involves identifying potential security threats through monitoring systems, user reports, or automated alerts. Security teams analyze anomalous behavior, suspicious network traffic, or system performance issues to confirm actual incidents.

Subsequently, Initial Assessment determines the incident's scope, severity, and potential impact. Analysts classify the threat level, identify affected systems, and estimate business disruption to prioritize response efforts appropriately.

Following assessment, Containment focuses on preventing further damage by isolating affected systems, blocking malicious network traffic, and implementing emergency security measures. Teams work quickly to limit the incident's spread while maintaining critical operations.

Next, Investigation involves detailed forensic analysis to determine attack vectors, compromised data, and threat actor methods. Specialists collect digital evidence, analyze logs, and document findings for legal and remediation purposes.

Then, Recovery includes restoring systems from clean backups, applying security patches, and validating system integrity. Teams gradually bring systems online while monitoring for residual threats or reinfection attempts.

Finally, Post-Incident Review evaluates response effectiveness, identifies improvement opportunities, and updates security policies. This learning phase strengthens future incident response capabilities and prevents similar breaches.
